lewk.

A lightweight, full-bleed CSS framework. One accent color, alpha-stepped grayscale, hairline rules instead of cards, and instant theme swapping. Right-click the swatches in the corner to roll a random theme.

Sidebar — hairline divider (gap: 0)

Asymmetric layout with a vertical hairline between columns. Unlike .split, uses gap: 0 — whitespace comes from padding on each column, so the border can meet adjacent horizontal rules at the corner. Modifiers: --right, --wide, --thin.

Sidebar — borders meeting (--ruled / .section--flush)

Two ways to make perpendicular hairlines meet at a corner rather than leaving a gap. --ruled: the sidebar owns its own border-top with no padding-top on the container — both borders originate at y=0 and meet. .section--flush: same effect when the parent .section provides the horizontal border; removes the section's padding-top so the child grid starts flush with it.

Margin note

Floats outside the prose column on wide viewports (≥1400px); falls back to an inline block with a left rule on narrower screens. Override --note-width (default: 12rem) on the element to resize.

Lewk is an editorial CSS framework with a single accent color and alpha-stepped grayscale. Notes appear in the right margin on wide viewports, where they hover beside the text they annotate without interrupting reading flow. Every other token derives via color-mix(), so the entire UI re-tints from one pair. Hairline rules instead of cards keep the layout open and typographic.

The theme system is designed so swapping a theme never breaks layout. Foreground and background swap cleanly, the accent shifts, and all swatches, rules, and buffers follow.

Themes

Themes set only --fg, --bg, and --accent. Every other token is derived via color-mix(), so the entire UI re-tints from one pair. Click a swatch in the lower right; right-click anywhere on the switcher to roll a random theme. All built-in pairs target WCAG AA (4.5:1) for body text.
